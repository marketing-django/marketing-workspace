# Project architecture

How the workspace is organized, and the tooling that supports it.

## Content layers

We organize information across four layers:

1. [Contributing](README.md): guidance on how to work within this workspace.
2. [Knowledge base](../knowledge-base/README.md): general-purpose information about Django, relevant for a wide range of projects.
3. [Marketing strategy](../marketing-strategy/README.md): information and working documents related to marketing specifically, across a wide range of projects.
4. Projects, like [Website redesign](../website-redesign/README.md): information about a specific project.

Each section folder uses a `README.md` as its landing page, so it also renders when browsing the folder on GitHub.

View [quality assurance](./quality-assurance.md) for more information on what should be in different layers.
